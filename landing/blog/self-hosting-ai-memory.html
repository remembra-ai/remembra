Self-Hosting AI Memory: Complete Guide to Running Your Own Remembra Server

Keep your AI memories on your own infrastructure. Full control, full privacy, no external dependencies. From Docker quickstart to production deployment.

AI memory contains sensitive information—your projects, preferences, decisions, and context. Many developers and organizations want this data on their own infrastructure, not in someone else's cloud.

Remembra is designed for self-hosting. This guide covers everything from a 30-second Docker quickstart to production-ready deployment with backups and monitoring.

Why Self-Host AI Memory?

  • Data sovereignty — Your memories never leave your infrastructure
  • Compliance — Meet GDPR, HIPAA, or internal security requirements
  • Air-gapped environments — Works fully offline
  • No recurring costs — One-time setup, free forever
  • Customization — Modify and extend as needed (Apache 2.0 license)

Quick Start (30 Seconds)

The fastest way to get self-hosted AI memory running:

docker run -d \
  --name remembra \
  -p 8787:8787 \
  -v remembra-data:/data \
  remembra/remembra

That's it. Your memory server is running at http://localhost:8787.

Verify It's Working

curl http://localhost:8787/health
# {"status":"ok","version":"0.9.9"}

Connect Your AI Tools

For Claude Code:

claude mcp add remembra -- npx -y remembra-mcp --url http://localhost:8787

For Cursor or other tools, add this to your MCP config:

{
  "mcpServers": {
    "remembra": {
      "command": "npx",
      "args": ["-y", "remembra-mcp"],
      "env": {
        "REMEMBRA_URL": "http://localhost:8787"
      }
    }
  }
}

What's in the Container?

Unlike some memory solutions that require separate vector databases, Remembra bundles everything:

  • Qdrant — Vector storage for semantic search (embedded)
  • BGE embedding model — Generates embeddings locally (no API calls)
  • SQLite — Metadata and entity storage
  • REST API — Full memory management interface

No external dependencies. No API keys needed. Fully offline-capable.

Production Deployment

For production use, you'll want persistent storage, resource limits, and proper configuration.

Docker Compose (Recommended)

Create a docker-compose.yml:

version: '3.8'

services:
  remembra:
    image: remembra/remembra:0.9.9
    container_name: remembra
    restart: unless-stopped
    ports:
      - "8787:8787"
    volumes:
      - ./data:/data
    environment:
      - REMEMBRA_LOG_LEVEL=info
      - REMEMBRA_MAX_MEMORIES=100000
    deploy:
      resources:
        limits:
          memory: 2G
        reservations:
          memory: 512M
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8787/health"]
      interval: 30s
      timeout: 10s
      retries: 3

Start with:

docker-compose up -d

Environment Variables

Key configuration options:

  • REMEMBRA_LOG_LEVEL — debug, info, warn, error (default: info)
  • REMEMBRA_MAX_MEMORIES — Maximum stored memories (default: 50000)
  • REMEMBRA_PORT — Server port (default: 8787)
  • REMEMBRA_AUTH_TOKEN — Optional API authentication token

Adding Authentication

For exposed deployments, add an auth token:

docker run -d \
  -p 8787:8787 \
  -v remembra-data:/data \
  -e REMEMBRA_AUTH_TOKEN=your-secret-token \
  remembra/remembra

Then include the token in your MCP config:

{
  "env": {
    "REMEMBRA_URL": "http://your-server:8787",
    "REMEMBRA_TOKEN": "your-secret-token"
  }
}

Deploying to a Server

Option 1: Any VPS (DigitalOcean, Hetzner, etc.)

Requirements: 1GB RAM minimum, Docker installed.

# SSH to your server
ssh user@your-server

# Install Docker if needed
curl -fsSL https://get.docker.com | sh

# Run Remembra
docker run -d \
  --name remembra \
  --restart unless-stopped \
  -p 8787:8787 \
  -v /opt/remembra:/data \
  remembra/remembra

Option 2: Coolify / CapRover

If you use a self-hosted PaaS:

  1. Create a new service with image remembra/remembra:latest
  2. Expose port 8787
  3. Mount a persistent volume to /data
  4. Deploy

Option 3: Kubernetes

apiVersion: apps/v1
kind: Deployment
metadata:
  name: remembra
spec:
  replicas: 1
  selector:
    matchLabels:
      app: remembra
  template:
    metadata:
      labels:
        app: remembra
    spec:
      containers:
      - name: remembra
        image: remembra/remembra:0.9.9
        ports:
        - containerPort: 8787
        volumeMounts:
        - name: data
          mountPath: /data
        resources:
          limits:
            memory: "2Gi"
          requests:
            memory: "512Mi"
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: remembra-pvc

Backups

Your AI memory is valuable. Set up automated backups.

Simple Backup Script

#!/bin/bash
# backup-remembra.sh
BACKUP_DIR=/backups/remembra
DATE=$(date +%Y-%m-%d)

# Stop container briefly for consistent backup
docker stop remembra

# Backup data directory
tar -czf $BACKUP_DIR/remembra-$DATE.tar.gz /opt/remembra

# Restart
docker start remembra

# Keep last 7 days
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

Add to cron for daily backups:

0 3 * * * /opt/scripts/backup-remembra.sh

Live Backup (No Downtime)

For zero-downtime backups, use the API export endpoint:

curl -o backup.json http://localhost:8787/api/export

Restore with:

curl -X POST -d @backup.json http://localhost:8787/api/import

Upgrading

Remembra uses automatic migrations. Upgrading is straightforward:

# Pull latest version
docker pull remembra/remembra:latest

# Stop current container
docker stop remembra

# Remove old container (data is in volume, safe!)
docker rm remembra

# Start with new version
docker run -d \
  --name remembra \
  -p 8787:8787 \
  -v remembra-data:/data \
  remembra/remembra:latest

Or with Docker Compose:

docker-compose pull
docker-compose up -d

Troubleshooting

Container won't start?

docker logs remembra

Check for port conflicts or permission issues on the data directory.

Out of memory?

The embedding model needs ~500MB RAM. Ensure your container has at least 1GB available.

Slow recall queries?

If you have >50k memories, consider increasing the memory limit or enabling the optional PostgreSQL backend for larger deployments.

Connection refused from MCP client?

  • Verify the server is running: docker ps | grep remembra
  • Check firewall rules allow port 8787
  • Test with curl: curl http://your-server:8787/health

Advanced: External Database

For large-scale deployments (>100k memories), you can use an external Qdrant cluster:

docker run -d \
  -p 8787:8787 \
  -e QDRANT_URL=http://your-qdrant-cluster:6333 \
  -e QDRANT_API_KEY=your-key \
  remembra/remembra

This separates compute from storage and allows horizontal scaling.
